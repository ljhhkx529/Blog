# 📦 知识库与文章归档

这是一个专用于存放和管理业务文档、博客文章及行业指南的纯 Markdown 仓库。

本仓库采用极简的结构进行知识管理，所有内容均以 `.md` 格式长期保存，方便在电脑端使用文本编辑器（如 VS Code、Typora）或在手机端通过 GitHub App 随时查阅与编写。

## 📂 目录结构 (Directory Structure)

为了保持文档的清晰易读，请按照以下目录分类存放文章：

- `📁 guides/` —— **业务指南** (例如：俄罗斯专线清关避坑指南、双清包税流程)
- `📁 news/` —— **行业动态** (例如：最新航运政策、运费调整通知)
- `📁 SOP/` —— **标准操作流程** (内部业务操作规范、客户对接话术)
- `📁 drafts/` —— **草稿箱** (正在构思、尚未完成的灵感和半成品)

## ✍️ Markdown 写作规范

为了在 GitHub 网页端和手机端获得最佳的阅读体验，建议遵循以下基础规范：

1. **文件名**：尽量使用全小写英文和连字符，例如 `russia-customs-2026.md`，避免使用中文文件名导致链接乱码。
2. **层级结构**：合理使用 `#` (一级标题)、`##` (二级标题) 划分文章结构。
3. **重点突出**：使用 `**加粗**` 标记核心结论或注意事项。
4. **表格与列表**：对于运费报价或流程步骤，优先使用 Markdown 的无序列表 `-` 或 `| 表格 |` 形式展现。

## 📱 移动端使用指南

1. 在手机上下载并登录 **GitHub App**。
2. 找到本仓库 (推荐将其设置为 `Star` 收藏，方便快速触达)。
3. 进入对应文件夹，点击右上角的 `+` 即可新建 `.md` 文件并随时记录灵感。
